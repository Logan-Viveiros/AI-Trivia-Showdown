**AI Trivia Showdown Design Document**

# **Project Concept**

**AI Trivia Showdown** is a multiplayer gaming experience where you progress by answering questions generator from an AI prompt. A host player chooses a game and writes a prompt and other players can then connect to the game and all answer questions as they play the game. Players can treat the game competitively, seeing who can get the highest score on the leaderboard, or like a fun, interactive studying app.

## **Game ideas**

- Classic (no game, just answering questions)  
- Tower Defense  
- Roguelite  
- Dungeon Crawler  
- Factory  
- PVP (or Battle Royale)  
- Deck Building Battles  
- Implement preexisting games with an API  
  - Certain games come with an API to track player data with programs. Using an API, the website can track when a player reaches certain milestones and then prompt a question.

## **Multiplayer**

AI Trivia Showdown will not have real-time synchronized multiplayer. The server will not update physics, movement, or game objects in real time. Each player will run the game logic locally on their client. The server will store player names, scores, lobby data, and game state. It will also be the endpoint for user interaction (players can still interact with each other, just not in real-time).

Multiplayer will consist of:

* Shared lobby system  
* Live leaderboard updates  
* Methods of interacting with other players  
* No real-time synced physics

# **Screens or States**

Most of the logic will be on two specific screens, the **Game** screen and the **Quiz** screen, however there are 7 total screens:

* **Menu** \- The menu is the first screen you see when you load up the website. You will have the option to either join a lobby or create a lobby, each leading to different screens. There may be an account system where you can earn a currency as you play games to unlock cosmetics in said games, though I’m unsure of that currently.  
* **Create Lobby** \- The create lobby screen is where you enter in the details of the game: the AI prompt for the quiz, the number of questions, what game to actually play, and when the game should end (score or time). Once you finish entering all the details, the player creating the lobby will click “Create Lobby” and enter the **Lobby** screen.  
* **Join Lobby** \- The join lobby screen will prompt you to enter a lobby code and a name to join a lobby, where you will then enter the **Lobby** screen.  
* **Lobby** \- The lobby screen will display the lobby code and all connected players. The creator of the lobby will also be given the option to start the game.  
* **Game** \- The game screen is where each player will play the game. Players will likely spend most of their time on this screen enjoying the game that the host picked. As they keep playing the game, they will be forced to answer questions on the **Quiz** screen to progress through the game.  
* **Quiz** \- The quiz screen will simply display a question and multiple answers, with only 1 answer being correct (all generated from AI). If they get the question correct, they will go back to the **Game** screen and progress in the game. If they get it wrong, they will either get a different question to answer or go back to the **Game** screen without progressing, depending on the game and whichever option makes the most sense with the game.  
* **Result** \- The result screen will appear once the end condition of the game has been met, and will show the final leaderboard to all players as well as every question they answered and their question accuracy.

# **User Interaction**

Very simple experience, there are only 4 interactions with the site:

* **Play The Game** \- The biggest interaction in the website is simply the player playing the game chosen by the host. The term “playing” is very broad, because there will be multiple games for the host to choose from, so it’s not possible to narrow exactly what the player will be doing while playing the chosen game.  
* **Answer Questions** \- The players will click 1 of (probably) 4 options to answer questions generated by AI to progress through the game.  
* **Create a Lobby** \- A host player will enter in all the details for a lobby: the game, quiz prompt, number of questions, and condition to end the game (score or time).  
* **Join a Lobby** \- Players will enter a code given by a host to join the host’s lobby.

# **Persistent Elements**

The centerpiece of the entire interaction is the game. Lobbies are created to play the game, questions are answered to progress the game, the game is the persistent element that determines all interactions. You can only progress the games by answering questions, motivating players to answer the questions to keep having fun (by playing the game). The game itself will get more difficult as you keep playing so as to not make the gameplay stagnant.
